# Modern TodoList

现代化的跨设备任务管理系统，基于艾森豪威尔矩阵的四象限任务管理法。

## 🚀 功能特性

### 核心功能
- 📝 **任务管理** - 完整的任务CRUD操作
- 🏷️ **清单分类** - 多清单管理，支持图标和颜色
- 🔍 **智能搜索** - 实时搜索任务内容、备注、责任人
- 🔄 **跨设备同步** - Vercel数据库实时同步
- 📱 **响应式设计** - 完美支持手机、平板、电脑

### 视图系统
- 📅 **今日视图** - 今日任务和逾期任务
- 📆 **本周视图** - 本周任务概览
- ⚡ **四象限视图** - 基于艾森豪威尔矩阵的智能分类
- 📋 **看板视图** - 按状态分列的任务看板
- 📊 **时间线视图** - 任务时间线可视化

### 任务属性
- 🎯 **优先级管理** - 高/中/低优先级标签
- ⏰ **重复规则** - 支持每日、每周、每月、每年重复
- 📎 **图片附件** - 任务关联图片文件
- 📝 **Markdown备注** - 富文本任务备注
- 🔗 **依赖关系** - 任务间依赖和前置条件

## 🛠️ 技术栈

- **前端框架** - 原生JavaScript + 模块化架构
- **构建工具** - Vite
- **数据库** - Vercel PostgreSQL
- **部署平台** - Vercel
- **UI风格** - 现代化黑色主题

## 🌐 部署

### Vercel部署
1. 在Vercel创建PostgreSQL数据库
2. 设置DATABASE_URL环境变量
3. 连接GitHub仓库到Vercel
4. 配置构建命令：`npm run build`
5. 部署完成！

## 🎯 使用方法

### 四象限任务管理法
- **第一象限**（🔥 重要且紧急）：立即处理
- **第二象限**（📋 重要不紧急）：计划处理
- **第三象限**（⚡ 紧急不重要）：委托他人
- **第四象限**（💤 不重要不紧急）：删除/搁置

### 快速开始
1. 创建任务时设置重要性和紧急性
2. 使用四象限视图查看任务分类
3. 在不同设备上登录查看实时同步

## 🔧 配置选项

### 环境变量
- `DATABASE_URL` - PostgreSQL数据库连接字符串
- `AUTH_TOKEN` - 用户认证token（可选）

### Vercel配置
- 构建命令：`npm run build`
- 输出目录：`dist`
- 开发命令：`npm run dev`
